## 5. 推荐维护策略

后续新增文档时，建议尽量落到下面三类之一：

- `知识文档`
  结论稳定、适合长期复用，例如架构、API、平台机制。
- `问题记录`
  针对一个具体 bug 或一类问题的排查与结论。
- `操作指南`
  面向“怎么做”的文档，例如发布、设置、调试。

如果以后继续整理，可以考虑第二步再做：

1. 给文件名前加前缀，例如 `知识-`、`问题-`、`指南-`
2. 合并明显重复的历史文档，尤其是 pin / 实时更新 / 重构计划这几组
3. 在每篇历史文档开头补一行“当前是否仍有效”
